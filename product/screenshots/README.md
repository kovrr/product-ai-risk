# Screenshots Folder

This folder contains screenshots for product requirements and design references.

## Naming Convention

Use descriptive names with dates:
- `YYYY-MM-DD_component-name_description.png`

### Examples:
- `2025-11-04_dashboard_new-layout.png`
- `2025-11-04_risk-register_card-view.png`
- `2025-11-05_compliance-readiness_wizard-mockup.png`

## Organization

You can create subfolders by component:
```
screenshots/
├── dashboard/
├── risk-register/
├── compliance-readiness/
├── ai-assurance-plan/
├── governance-monitoring/
├── integration-hub/
├── financial-quantification/
└── assets-visibility/
```

## How to Reference in PRD

In the PRD.md file, reference screenshots like this:

```markdown
**Design Reference**: `product/screenshots/2025-11-04_dashboard_new-layout.png`
```

Or for organized folders:

```markdown
**Design Reference**: `product/screenshots/dashboard/new-layout.png`
```

## Tips

- ✅ Use high-resolution screenshots
- ✅ Annotate with arrows/text if needed
- ✅ Include "before" and "after" for changes
- ✅ Capture full screen or specific components
- ✅ Name files descriptively
