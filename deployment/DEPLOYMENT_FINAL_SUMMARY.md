## üîÑ News Articles Sync Logic

### What Happens to Articles:

**Before Deployment:**
- Local DB: 20+ articles (your curated collection)
- Server DB: 23 articles (20 seed + 3 from crawler)

**During Deployment:**
- ‚ùå Server DB: **DROPPED** (all 23 articles deleted)
- ‚úÖ Server DB: **RECREATED** (fresh database)
- ‚úÖ Import: 20 latest articles from local DB
- ‚úÖ Crawlers run: Fetch new articles from IAPP

**After Deployment:**
- Server DB: 20+ articles
  - 20 from local DB (latest by published_at)
  - 0-3 new from IAPP crawler (if any new articles)

### Why Drop Database?

**Pros:**
- ‚úÖ Always get fresh data from local
- ‚úÖ No stale/duplicate articles
- ‚úÖ Clean state every deployment
- ‚úÖ Matches local DB structure exactly

**Cons:**
- ‚ùå Loses crawler-fetched articles between deployments
- ‚úÖ **BUT**: Crawler runs after deployment to re-fetch
